# 公众号文章终审质检清单

> 基于《得到品控手册11.0》金线品控（最高标准）+ 红线品控（绝对底线），适配「当车学会走路」系列公众号文章场景。

---

## 第一层：红线品控（绝对底线，一票否决）

> 以下任何一项不通过，文章**禁止发布**。

### R1 事实错误
- [ ] 所有关键数据（市场规模、增长率、时间、公司名、产品名）**与事实一致**
- [ ] 没有编造信息或无依据的断言
- [ ] 引用的行业案例**真实发生过**，不是臆造

### R2 违规与敏感内容
- [ ] 不涉及法律法规禁止的内容（涉黄、涉暴、政治敏感等）
- [ ] 不违反广告法（无虚假宣传、无夸大承诺）
- [ ] 不侵犯他人版权（图片、文字、数据均合规）

### R3 误导性内容
- [ ] 不故意误导读者（如夸大效果、隐瞒风险）
- [ ] 不使用"震惊体""贩卖焦虑体"等诱导点击
- [ ] 标题准确反映文章内容，不做标题党

### R4 抄袭与低质量
- [ ] 核心内容为**独立产出**，不是复制粘贴
- [ ] 引用内容**标注来源**
- [ ] 内容不空洞、不重复、不是"正确的废话"

### R5 商业机密泄露
- [ ] 所有公司内部案例经过**脱敏处理**
- [ ] 不泄露未公开的商业数据、项目细节、内部决策

---

## 第二层：金线品控（最高标准，追求卓越）

> 以下为文章应努力达到的高质量标准，逐项检查。

### J1 准确性（金线级）
- [ ] 关键信息经过**多源交叉验证**（官方公告、论文、财报）
- [ ] 不使用"大概""可能""据说"等模糊表述覆盖关键事实
- [ ] 专业术语使用**准确**，符合行业主流定义
- [ ] 行业素材引用**一手来源**，拒绝二手转载

### J2 深度性（金线级）
- [ ] 超越表层信息，提供**独到分析或底层逻辑拆解**
- [ ] 不是简单罗列"XX公司做了什么"，而是解释"**为什么这样做**"
- [ ] 常规解释部分：理论框架完整、逻辑推导严密、结论可验证
- [ ] 独到洞见部分：在常规解释基础上形成**认知突破**，改写叙事结构
- [ ] 洞见经过作者与AI的**多轮质疑-深化-确认**，非敷衍了事

### J3 用户价值（金线级）
- [ ] 读者读完有**明确收获**——新知、新视角或新判断框架
- [ ] 明确解决了读者的某个**认知痛点**
- [ ] 读者能**转述给别人**——有"可以带走的一句话"
- [ ] 价值超出读者预期（不只是"有道理"，而是"原来如此"）

### J4 专业性（金线级）
- [ ] 文章体现了**只有作者能提供的信息**（实战经验、一手案例、独家视角）
- [ ] 知识体系严谨，没有常识性错误
- [ ] 与市面已有内容有**明确差异化**

### J5 结构清晰（金线级）
- [ ] 文章有**一条清晰的主线**，每段都在推进
- [ ] 三段式结构（现象→常规解释→洞见）**边界清晰**
- [ ] 叙事结构选择得当（线性/总分/对比/递进/循环）
- [ ] 段落**结论先行**（总-分），不铺垫半天才到重点
- [ ] 段落长度100-300字，无超过500字的巨型段落

### J6 语言流畅（金线级）
- [ ] **口语化书面语**——书面但不生硬，通俗但不口水
- [ ] 不使用"赋能""抓手""底层逻辑"等过度互联网化词汇
- [ ] 句子不超过40字，无连续3个从句的长句
- [ ] 长短句交替，有阅读节奏感
- [ ] 技术概念有**直觉化解释**，零专业背景读者也能理解
- [ ] 类比/比喻是**辅助理解工具**，不误导、不冗长、风格统一

### J7 原创性（金线级）
- [ ] 核心观点、数据解读、案例分析为**独立产出**
- [ ] 反面案例和局限条件也被正视，不做选择性呈现
- [ ] 预判并回应了读者可能的质疑

### J8 时效性（金线级）
- [ ] 引用的行业数据**不超过1年**
- [ ] 不使用过时案例佐证当前趋势
- [ ] 技术发展动态基于最新公开信息

### J9 合规性（金线级）
- [ ] 竞品对比时**客观呈现双方优劣势**，不主观攻击
- [ ] 不贬低具体公司或个人
- [ ] 符合平台发布规则

## 第三层：公众号专项检验

> 针对公众号场景的专项检查。

### G1 标题
- [ ] 不超过20个字
- [ ] 包含具体信息点（数字、反常识、悬念）
- [ ] 在信息流中**1秒内可识别**
- [ ] 不使用读者不熟悉的专业术语

### G2 开头钩子
- [ ] 前200字制造**认知冲突、悬念或代入感**
- [ ] 不以"随着XX的发展"等套话开头
- [ ] 从钩子到正文过渡**自然**
- [ ] 开头提出的问题在正文有**完整回应**

### G3 结尾
- [ ] 回应开头，形成**闭环**
- [ ] 提升一个认知维度，不是简单重复
- [ ] 系列文章结尾有**下一篇预告**但不剧透
- [ ] 不说教、不煽情、不使用"让我们拭目以待"等空洞收束
- [ ] 不在结尾引入新概念

### G4 字数与排版
- [ ] 单篇**2000-3000字**（首篇可放宽至3500字）
- [ ] 小标题本身就是信息点
- [ ] 关键结论/核心数据**加粗**
- [ ] 段落间有空行，视觉留白

### G5 阅读体验
- [ ] 技术密集段落后有**缓冲段落**（类比、故事、提问）
- [ ] 不连续超过3段同一类型内容
- [ ] 有"你"的意识——经常从读者视角提问和回应
- [ ] 每个技术段落后回答读者心中的**"那又怎样？"**

---

## 第四层：系列一致性检验

> 确保系列文章的整体连贯性。

### S1 暗线植入
- [ ] 五柱主线（世界模型、通用平台、ODD、Fail Operation、数据闭环）植入**自然不刻意**
- [ ] 暗线以故事/现象/类比形式出现，**不以框架命名**
- [ ] 读者不需要知道五柱框架也能**独立理解单篇**

### S2 术语与类比
- [ ] 同一概念**前后表述一致**
- [ ] 类比风格**统一**，不频繁切换

### S3 作者人设
- [ ] 每篇都是同一个"泊车工程师"在说话
- [ ] 前后文**没有自相矛盾**的事实或观点

---

## 审稿流程

### Step 1：红线扫雷（5分钟）
仅过第一层（R1-R5），任何一项不通过 → **打回重写**。

### Step 2：通读全文（10分钟）
不逐项检查，感受整体节奏、阅读体验和信息密度。

### Step 3：金线逐项（20分钟）
过第二层（J1-J9）+ 第三层（G1-G5），标注修改建议。

### Step 4：系列一致性（5分钟）
过第四层（S1-S3），与前文交叉检查。

### Step 5：修改后二审
只检查被修改的部分是否引入新问题。

### Step 6：终审通过 → 排版发布

### 评分标准
| 等级 | 含义 | 处理方式 |
|------|------|---------|
| ✅ 通过 | 完全达标 | 无需修改 |
| ⚠️ 微调 | 小问题，不影响整体 | 建议修改 |
| ❌ 修改 | 结构性问题 | 必须修改后再审 |
| 🚫 否决 | 红线硬伤 | 打回重写 |
