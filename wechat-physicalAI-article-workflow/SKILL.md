# 公众号文章协作写作工作流

## 系列定位

- **系列名称**：「当车学会走路」— L4智驾与具身智能的技术共振
- **目标读者**：泛科技爱好者
- **作者优势**：奔驰Park2Park/记忆泊车量产实战 + L4泊车系统架构 + ISO 26262/SOTIF
- **内容风格**：技术 + 行业洞察结合，场景化叙事

## 五柱技术主线

贯穿全系列的隐形认知架构（不以框架命名出现）：

1. **世界模型** — 理解物理世界与预测未来（System 1 + System 2）
2. **通用硬件平台** — 大脑层（计算/AI栈趋同）+ 躯体层（执行器标准化）
3. **ODD** — 世界模型理解力的边界 = 系统能力边界
4. **Fail Operation** — 脑干级硬约束 + System 2 降级策略
5. **数据闭环** — 仿真 + 真实数据飞轮，驱动世界模型迭代
6. **VLA** — 对照路线，衬托世界模型的深度理解力

## 三段式文章结构

每篇文章遵循以下结构：

### 第一段：现象/问题（作者主导）
- 抛出一个读者"说起来都知道，但讲不清楚"的现象或问题
- 选题由作者与AI共同探讨，**最终由作者敲定**
- 参考已有15篇文章规划来确定选题顺序和暗线分布

### 第二段：常规解释（AI主导）
- 基于当前最新主流认知的详细分析
- 搭建基础理论框架、给出逻辑推导过程、得出主流结论
- 要求深入调研，引用一手来源
- 目标：让读者觉得"分析得有道理"，但感受到"食之无味"

### 第三段：独到洞见（作者主导，AI辅助深化）
- 在常规解释基础上，提供作者独有的认知突破
- 可能完全改写解释框架和叙事结构
- 让读者有"耳目一新、真正得到"的感觉

**AI在此阶段的职责：**
1. 基于常规解释的分析，向作者提供**洞见方向的参考建议**（如"从XX角度看可能会得出不同的结论"、"这个现象的底层逻辑其实是YY"）
2. 作者提出洞见后，AI帮助**拓展、深化、润色**，将模糊的想法变成精准的表达
3. **质疑与引导**：如果作者提出的洞见不够清晰或缺乏认知突破性，AI必须主动质疑并引导作者进一步思考。例如："这个洞见和常规解释的区别是什么？读者读完会有'原来如此'的感觉吗？" "这个观点是否只是对主流结论的重述，而不是认知升级？"AI不是盲目执行，而是充当"苏格拉底式的提问者"

## 协作分工

| 角色 | 职责 |
|------|------|
| 作者（主公） | 敲定选题、提供洞见、实战经验素材、终审拍板 |
| AI（诸葛亮） | 行业调研、常规解释撰写、洞见拓展润色、品控终审 |

## 写作流程

1. **选题对齐**：基于系列规划确定本期选题 → 作者确认
2. **常规解释初稿**：AI调研主流认知 → 撰写理论框架+逻辑推导+结论
3. **洞见融入**：作者提供独到洞见 → AI拓展润色，融入文章
4. **品控终审**：AI按质检清单逐项审核 → 给出修改建议 → 作者确认修改
5. **定稿发布**

## 质检标准

终审时按质检清单逐项过，采用四层审查机制：
1. **红线品控**（R1-R5）：绝对底线，一票否决
2. **金线品控**（J1-J9）：最高标准，追求卓越
3. **公众号专项**（G1-G5）：场景适配
4. **系列一致性**（S1-S3）：跨篇连贯

清单详见 [references/质检清单.md](references/质检清单.md)。
