# AI Daily Digest Skill

## 概述

本 Skill 用于生成每日 AI 领域简报，覆盖智能驾驶、具身智能、AI 大模型三大领域，同时关注行业前沿洞见。

## 三大领域

### 1. 智能驾驶 (Intelligent Driving)
- Tesla, Waymo, 小米SU7, 华为ADS, 蔚来, 小鹏, 比亚迪以及其它有亮点的新兴玩家等
- 自动驾驶算法、芯片、传感器、无人出租车商业化、L3/L4 等

### 2. 具身智能 (Embodied AI)
- 人形机器人、机械臂、物理AI、四足机器人、机器人平台等
- Boston Dynamics, Figure AI, 宇树, 特斯拉Optimus 以及其它有亮点的新兴玩家等

### 3. AI 研究应用 (AI Research)
- 新模型发布、论文突破、新 AI 应用、Agent、工作流等
- OpenAI, Anthropic, Google DeepMind, Meta AI, DeepSeek, Minimax, 智谱清言以及其它有亮点的新兴玩家等
- arxiv 最新论文

## 行业洞见来源

除了官方新闻，还需关注：

| 类型 | 示例 |
|------|------|
| 行业领袖 | Yann LeCun, Jensen Huang, Sam Altman, Demis Hassabis |
| 科技博主 | 热门 AI Newsletter、YouTube 技术博主 |
| 专栏作家 | Wired, The Verge, TechCrunch 等科技媒体记者 |
| 产品专家 | 各大厂产品经理、技术负责人 |
| 学术前沿 | 顶级实验室科学家、教授 |

**搜索渠道**：X/Twitter、LinkedIn、Substack、个人博客、学术社交网络

## 信源优先级

1. **官方一手**：官方博客、GitHub、arxiv、论文、公司公告 → 优先收录
2. **原始内容**：CEO/科学家社交媒体原始帖子、访谈 → 优先收录
3. **权威分析**：知名科技记者/博主的深度分析、行业报告 → 可收录，需确认分析文章本身的发布日期 ≤ 24h

**拒绝**：无法确认发布日期的来源、无来源的二道手转发、综合回顾/盘点类文章

**聚合站处理**：聚合站文章仅作为发现线索，必须追溯到原始信源，确认原始信源发布日期 ≤ 24h 后方可收录。

## ⏱ 逐条时间验证流程

生成日报时，每条候选新闻都必须经过以下验证：

1. **搜索阶段**：使用 `freshness=day` 过滤，缩小候选范围
2. **抓取阶段**：对每条候选，用 `web_fetch` 抓取页面，检查 `<meta>` 标签（`datePublished`、`article:published_time`、`pubdate`）或页面正文中的日期
3. **判断阶段**：确认该日期在当前时间 ±24h 内
4. **剔除阶段**：无法确认日期的 → 排除；日期超过 24h 的 → 排除

**搜索引擎的 freshness 过滤不完美**，会返回略超 24h 的结果，甚至某些聚合文章会将旧事件重新包装为近期内容。因此每条都必须独立验证，不能仅依赖搜索 API 的时间过滤。

**注意**：信源内容中提到的「未来预期」或「历史回顾」不算超时——只有信源本身的发布时间需要在 24h 内。例如：一篇 4/18 发布的分析文章讨论「DeepSeek V4 预计四月下旬发布」是完全合规的，因为信源发布日在 24h 内。

## 输出格式

简报需包含：
- 日期和领域分类
- 每条资讯 2-3 句话核心要点
- 标注来源链接
- 行业洞见单独板块

## 使用方式

本 Skill 通常通过 Cron 定时任务触发，自动发送到指定渠道。

如需手动触发：
```
生成今日 AI 日报
```
